AI → GitHub → Live website

From a prompt to a published page.

Every step from asking an AI to build your HTML, to creating a GitHub repository, to deploying it as a free public website. No command line — it all happens in your browser. Tick each step as you go.

0 / 14 done~ 20 minutesCost: free
01

Get the HTML from an AI

Build
Gather your raw content.
Your CV text, photos and links. The clearer the input, the better the result.
Write a prompt with three parts: what the page is, what content goes on it, and the look you want.
e.g. "A personal profile page. Here are my details: […]. Make it clean and modern with a deep green palette."
Add one technical line that saves trouble later: ask for a single self-contained HTML file.
"Give me one self-contained .html file with images embedded and fonts loaded from a CDN." Now the whole site is one file — nothing loose to manage.
Iterate until you're happy, then save the file to your computer.
"Make the header bigger," "use a warmer accent," "add a contact button" — repeat as needed.
02

Prepare the file

1 step
Rename the file to exactly index.html.
GitHub Pages serves this filename as your homepage. This single detail trips most people up.
TIP
Your profile page is already named index.html — you can skip straight to Phase 3.
03

Create the GitHub account & repository

Set up
Make a free account at github.com (skip if you have one).
Your username becomes part of your web address — pick something clean like your name.
Click the + at the top-right → New repository.
Name it yourusername.github.io for the cleanest possible URL.
Set it to Public, leave "Add a README" unticked, then click Create repository.
04

Upload your file

2 steps
Click "uploading an existing file" (or Add file → Upload files).
Drag index.html in, then click Commit changes.
It must sit in the root of the repo — not inside a folder.
05

Deploy with GitHub Pages

Publish
Open Settings → Pages (left sidebar of the repository).
Source: Deploy from a branch. Set branch to main, folder to / (root), then Save.
06

Find your link & verify

Done
Wait ~1 minute, refresh the Pages screen.
A banner appears: "Your site is live at…" with a Visit site button.
Open the link to confirm it works.
A 404 at first is normal — the first build takes a moment. If it's still blank after 5 min, check the file is named index.html at the repo root.
URL
Your address is predictable: https://yourusername.github.io if you named the repo that way, otherwise https://yourusername.github.io/repo-name/.
UPDATE
To change the site later, upload a new index.html to the same repo — it replaces the old one and goes live within a minute or two.

That's a live website.

Tick the last box and you're published to the public web.
